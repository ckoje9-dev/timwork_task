# Timwork — 건설 도면 탐색 인터페이스

건설 현장의 공종별·리비전별 도면을 탐색하고, 도면 위에 이슈를 기록하여 협업을 돕는 프로토타입 UI입니다.

---

## 실행 방법

```bash
npm install
npm run dev
```

브라우저에서 `http://localhost:5173` 접속

---

## 기술 스택

- **React 18** + **TypeScript** — UI 및 타입 안전성
- **Vite** — 빌드 도구
- **React Router v6** — SPA 라우팅 (탭 전환, URL 공유)
- **Zustand** — 전역 상태 관리 (localStorage 영속)
- **Tailwind CSS v3** — 유틸리티 기반 스타일링, 디자인 토큰 중앙 관리
- **Lucide React** — 아이콘
- **Google Maps API** — 대시보드 현장 위치 지도

## 핵심 기능 — 도면 탭

과제의 최종 목표인 **도면 탐색**, **도면 표시**, **컨텍스트 인식** 세 가지 필수 기능을 중심으로 설계되었습니다.

### 1. 도면 탐색

**설계 의도**

현장 소장 김철수 씨는 건물 위치보다 공종 단위로 생각하며 도면을 찾습니다. "건축 도면 전부", "설비 공종 최신 리비전"처럼 공종이 1차 탐색 축입니다. 좌측 트리를 공종 기준 아코디언으로 구성하여 실무 탐색 흐름과 일치시켰습니다.

**사용법**

1. 좌측 트리에서 공종 탭(건축·구조·설비·소방 등)을 펼칩니다.
2. 도면명을 클릭하면 뷰어에 즉시 표시됩니다.
3. 상단 검색창으로 도면명 키워드 검색, 공종 필터 버튼으로 빠른 좁히기가 가능합니다.
4. Region이 있는 도면(101동 구조)은 트리에서 `확대평면도A`, `확대평면도B`처럼 구역별 노드가 독립적으로 생성됩니다.

---

### 2. 도면 표시

**설계 의도**

현장 실무에서 도면은 세밀한 치수와 주석이 담긴 고해상도 이미지입니다. 축소된 썸네일이 아니라 확대·이동하며 직접 도면을 읽을 수 있어야 합니다. 비교 모드에서는 여러 공종 도면을 색상 레이어로 겹쳐 공종 간 간섭 여부를 한 화면에서 확인할 수 있습니다.

**사용법**

| 조작 | 방법 |
|------|------|
| 줌 인/아웃 | 마우스 휠 또는 우측 하단 줌 버튼 |
| 이동 | 드래그 (grab 커서) |
| 화면에 맞추기 | 우측 하단 **Fit** 버튼 |
| 리비전 전환 | 상단 헤더 드롭다운 (REV1 → REV2 → REV3) |
| 도면 비교 | 상단 **비교** 버튼 → 공종별 레이어 패널에서 투명도·가시성 조절 |
| 비교 화면 캡처 | 비교 모드에서 우측 하단 **카메라** 버튼 → PNG 저장 |

---

### 3. 컨텍스트 인식

**설계 의도**

수십 개의 도면을 오가다 보면 "지금 어떤 도면의 어떤 리비전을 보고 있는지" 잊기 쉽습니다. 상단에 항상 현재 도면 컨텍스트를 표시하여 혼란을 방지하고, 최근 열람·북마크 기능으로 자주 쓰는 도면에 빠르게 접근할 수 있게 했습니다.

**사용법**

- **브레드크럼** (상단): `프로젝트명 / 공종 / 도면명 / 리비전` 형태로 현재 위치 표시
- **리비전 드롭다운**: 각 리비전의 발행일·변경 요약을 보며 전환
- **북마크** (트리 항목 우클릭 또는 헤더 아이콘): 자주 보는 도면을 대시보드 북마크 탭에서 바로 열기
- **최근 항목** (대시보드): 최근 열람한 도면·이슈 최대 7개, 클릭 시 바로 이동

---

### 4. 폴리곤 기반 공간 탐색

**설계 의도**

`metadata.json`의 `position.vertices` (상위 도면 픽셀 좌표)와 `polygon.vertices` (공종 커버리지 좌표)를 SVG 오버레이로 시각화하여, 도면 트리 탐색과 별도로 **공간적 위치 인식** 기반의 탐색 경로를 제공합니다. "101동이 전체 배치도 어디에 있는지", "A구역이 평면도 어디에 해당하는지"를 클릭 한 번으로 탐색할 수 있습니다.

**사용법**

| 도면 | 폴리곤 클릭 동작 |
|------|----------------|
| **전체 배치도 (00)** | 건물 영역(101동·주민공동시설·주차장) 클릭 → 해당 도면으로 이동 |
| **101동 지상1층 평면도 (01, 건축)** | A구역·B구역 폴리곤 클릭 → 구조 확대평면도A/B로 이동 |

**주석 오버레이 (주석 토글)**

- 상단 헤더의 **주석** 버튼으로 폴리곤 오버레이와 이슈 핀을 일괄 표시/숨김
- 비교 모드 진입 시 자동으로 숨겨져 도면 가독성 확보
- 표시 레이어:

  | 레이어 | 색상 | 의미 |
  |--------|------|------|
  | 공종 외곽선 | 회색 점선 | 해당 공종 도면의 전체 커버리지 |
  | 구역 경계 | 색상 점선 | 도면 내 A/B 분할 구역 |
  | 리비전 범위 | 파란 점선 + 연한 채우기 | 현재 리비전의 변경 구역 |

- [x] 최근 항목 탭 (최근 열람 도면·이슈 최대 7개)
- [x] 북마크 탭 — 도면·이슈 북마크 목록, 클릭 시 바로 이동

## Mock API 구조

`src/api/client.ts`에서 한 줄만 변경하면 실제 백엔드로 전환됩니다:

```ts
// 변경 전 (Mock 모드)
const USE_MOCK_API = true;

// 변경 후 (실제 API 모드)
const USE_MOCK_API = false;
```

`.env` 파일에 API 주소 설정:

```env
VITE_API_BASE_URL=https://api.timwork.kr/v1
```

| 기능 | Mock 방식 | 실제 API 경로 |
|------|-----------|--------------|
| 도면 트리·메타데이터 | `/metadata.json` fetch + 가공 | `GET /drawings/tree` |
| 이슈 CRUD | 인메모리 배열 | `GET/POST/PUT/DELETE /issues` |
| 이슈 통계 | 인메모리 집계 | `GET /issues/stats` |
| 대시보드 | Mock JSON | `GET /dashboard` |
| 이슈 핀 | 클라이언트 전용 | 백엔드 협의 필요 |
